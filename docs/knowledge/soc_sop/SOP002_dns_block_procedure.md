# SOP-002: DNS 封锁操作流程

## 目的
规范对确认恶意的 DGA 域名实施 DNS 层面封锁的操作流程。

## 适用范围
经 SOP-001 分诊确认为中危及以上的 DGA 域名。

## 前置条件
- 已完成 SOP-001 告警分诊
- 已确认域名非白名单误报

## 流程步骤

### 1. 封锁准备
- 整理待封锁域名列表（含通配符规则）
- 确认封锁范围（单域名/整个 DGA 家族预测列表）
- 获取变更审批（紧急情况可事后补批）

### 2. 执行封锁
- 在 DNS 防火墙/RPZ 中添加封锁规则
- 配置 Sinkhole 将域名解析到受控 IP
- 在 Web 代理中同步添加域名黑名单
- 记录封锁操作的时间和操作人

### 3. 验证确认
- 测试封锁规则是否生效
- 确认 Sinkhole 日志正常记录
- 检查是否影响正常业务域名解析

### 4. 持续维护
- 每周审查封锁列表有效性
- 对已失效的 C2 域名定期清理
- 将封锁规则同步至所有 DNS 节点
