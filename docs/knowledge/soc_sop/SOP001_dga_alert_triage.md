# SOP-001: DGA 告警分诊流程

## 目的
规范 SOC 分析师对 DGA 检测告警的初始分诊和处置流程。

## 适用范围
所有由 DGA 检测引擎产生的告警事件。

## 流程步骤

### 1. 告警接收与确认（5 分钟内）
- 确认告警来源（DNS 日志/流量检测/端点检测）
- 记录告警时间、源 IP、涉及域名列表
- 查看告警置信度评分

### 2. 初步分析（15 分钟内）
- 在威胁情报平台查询涉及域名的信誉
- 检查域名是否命中已知 DGA 家族特征
- 确认源 IP 对应的资产和用户信息
- 排除已知白名单域名（CDN、云服务等）

### 3. 严重性判定
- **高危**: 域名已解析到活跃 C2 IP，或匹配已知恶意家族
- **中危**: 大量 NXDOMAIN 响应，疑似 DGA 扫描
- **低危**: 少量可疑域名，可能为误报

### 4. 处置决策
- 高危：立即执行 SOP-003 主机隔离
- 中危：执行 SOP-002 DNS 封锁并持续监控
- 低危：记录并加入观察列表，48 小时后复查
