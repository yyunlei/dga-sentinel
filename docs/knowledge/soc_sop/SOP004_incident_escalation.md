# SOP-004: 事件升级流程

## 目的
规范 DGA 相关安全事件的升级判定标准和流程。

## 适用范围
所有需要从 L1 分析师升级至 L2/L3 或管理层的 DGA 安全事件。

## 升级判定标准

### 升级至 L2 分析师
- 单主机检测到已知 DGA 家族的活跃 C2 通信
- 发现新型未知 DGA 模式，现有规则无法分类
- 告警涉及关键业务系统或高权限账户

### 升级至 L3/事件响应团队
- 多台主机同时出现 DGA 告警（疑似横向移动）
- 确认存在数据外泄行为
- 涉及勒索软件家族（如 CryptoLocker）的 DGA 活动

### 升级至管理层/CISO
- 影响核心业务系统可用性
- 涉及客户数据泄露风险
- 需要启动应急响应预案

## 升级流程
1. 填写事件升级报告（含时间线、影响范围、已采取措施）
2. 通过工单系统提交并电话通知接收方
3. 确认接收方已接手事件处置
4. 在工单中持续更新事件进展
5. 事件关闭后 5 个工作日内完成复盘报告
